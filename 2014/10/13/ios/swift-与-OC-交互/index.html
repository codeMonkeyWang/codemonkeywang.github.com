swift 与 OC 交互

1 开始

1.1导入

  • import Foundation
    • 重映射OC 类型到swift 中的同等类型 如id 到AnyObject
    • 重映射OC 核心类到Swift 中的替代类型，NSString 到 String
    • 重映射确定的OC概念到Swift概念,pointers到optionals
  • Swift 申明它的API作为Swift modules，这些modules 会生成OC的头文件，这些头文件映射回OC。一些 Swift api不映射回OC，因为他们取舍了语言特性发现这些在OC不可以。

互用性

  • OC Class初始化

    • OC 的init initWith 前缀会被去除 ，剩下的部分变小写并会成为第一个参数名

  • 访问属性

    • 使用点语法
    • 只有使用@property关键字声明的属性才能访问
  • id 兼容性
    • AnyObject 代替 id
  • 扩展
    • 你不能使用扩展来覆盖 Objective-C 类型中存在的方法与属性。
  • 闭包
    • blocks 会被自动导入为Swift的闭包 
- Swift 中变量的默认行为与 Objective-C 中 __block 变量一致。 (oc 中block只有添加__block 变量才可更改)
  • 对象比较
    • ==操作符会调用 NSObject 的 isEqual: 方法，想要比较内容需要实现自定义 isEqual 方法
    • 如果你希望类能够作为字典的键，需要实现Hashable协议实现hashValues属性
  • @objc 可以让你的Swift API在OC中使用
