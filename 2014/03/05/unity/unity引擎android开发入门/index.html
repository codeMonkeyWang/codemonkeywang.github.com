记录一下Unity的Android开发需要注意的事项。

使用Handheld类可以调用Android或ios 震动和指示灯还有播放全屏电影。

Unity Android 和桌面 Unity 有何不同

  • 强类型的 JavaScript
    出于性能原因，在 Unity Android 中，Javascript 动态类型始终关闭，有如将 #pragma strict 自动应用到所有脚本。
  • 建议最好使用 ETC 作为纹理压缩格式 （ ETC1/RGBA16 为默认是个）
    虽然 Unity Android 不支持 DXT/PVRTC/ATC 格式的纹理，如果使用的特定设备不支持这些压缩方法，Unity 会在运行时解压缩纹理到 RGB(A) 格式。这样可能影响 GPU 的渲染速度，所以推荐使用 ETC 格式替代。ETC 已作为 Android 2.0 之后所有版本的标准压缩格式。ETC 不支持 alpha 通道，考虑到必须使用 alpha 的大小、质量和渲染速度，RGBA 16-bit 将是最好的选择。

Android Remote 允许你使用设备远程控制 Unity 中的工程。

使用感受：

* 确实很方便
* 有一些支持还是不过，比如说震动就无法再手机上体现出来。

Android 还不支持的功能

  • 图形
    ETC 格式不支持非正方形纹理。
    不支持影片纹理，可以用全屏幕流媒体播放代替。请参阅影片回放页面了解更多信息。
  • 脚本
    Android 不支持 OnMouseEnter、OnMouseOver、OnMouseExit、OnMouseDown、OnMouseUp 和 OnMouseDrag 事件。
    不支持鸭子类型 (Duck Typing) 等动态类型。脚本使用 #pragma strict 将迫使编辑器将动态功能报告为错误。
    不支持通过 WWW 类的视频数据流。

分离应用程序二进制 (.OBB) 支持

  • 在播放器设置 (Player Settings) | 发布设置 (Publishing Settings) 可以看到将应用程序二进制 (.apk) 分离为扩展文件 (.apk + .obb) 的选项。

  • 仅在将大于 50 MB 的应用程序发布至 Google Play 商店时需要这一原理。
    详细可查阅http://docs.unity3d.com/Documentation/Manual/android-OBBsupport.html

反盗版检查

通过删除 AppStore DRM 保护来破解应用程序，并使其变成免费的的应用，这一现象非常普遍。Unity 配备了反盗版检查，可确定在提交至 AppStore 之后，应用程序是否经过修改。

可通过 Application.genuine 属性检查应用程序是否为正版（非盗版）。如果属性的返回值是 false，那么可以通知应用程序的用户，他/她正在使用破解的应用程序，或者可以禁止其访问应用程序上的某些功能。
注意： Application.genuineCheckAvailable 应与 Application.genuine 一并使用，以确认应用程序的完整性可得到实际验证。访问 Application.genuine 属性是一项相当昂贵的操作，不应用于帧更新期间或其他时序要求严格的代码。
