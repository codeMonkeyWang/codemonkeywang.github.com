命令

init 初始化

hexo init [folder]

new 新建

hexo new [layout] <title> hexo n [layout] <title>

  • 创建草稿 hexo new draft <title>
    • 存放于 source/_drafts路径
  • layout: scaffolds文件夹下的模板
    • 有2个默认layout post 和page
      • post 会保存在 source/_posts
      • page 会保存在 source
  • Alias: n
  • 参数
    • -p``-path 指定文章的存放位置(文件名也会被同时指定)
    • 例：hexo n "test" -p /path1/aaa

generate 生成

hexo generate hexo g

  • Option
    • -d --deploy 生成后部署
    • -w --watch 查看文件修改

publish 发布

hexo publish [layout] <filename>
hexo p [layout] <filename>

  • 发布一个草稿

server 本地服务

hexo server hexo s

  • Option 选项
    • -p, --port 覆盖默认端口
    • -s, --static 只显示静态文件，不会动态显示更改
    • -l, --log Enable 启用日志记录器
    • -d, --drafts 本地服务显示草稿

deploy 部署

hexo deploy hexo d

  • --setup
  • -g, --generate 先重新生成静态文件，然后部署
  • -m, --message 显示自定义消息提示

clean

hexo clean

  • 清除db.json缓存文件和public文件夹下的文件

list

hexo list routes

  • 列出hexo的目录下的所有文件

version

hexo version

更多请参考官方文档
