Atom安装Nuclide后编辑器卡死的解决办法

By 程序猿旺旺

发表于 Nov 2 2015

最近在学习React以及React Native,顺便试用了一下Facebook 基于Atom开发的IDE Nuclide，具体细节可以参考官网介绍(自备梯子)。

按照官网推荐的方法通过Atom的插件直接安装之后发现，存在很严重的问题–安装完之后打开atom的几乎是无法使用的，操作几下就出现严重的卡顿或卡死状态。

我的开发环境：
Mac OS x 10.11.1
Atom 1.1.0
nuclide-installer 0.0.32

我去nuclide在github上的开源仓库看来一下，发现好多人遇到类似的问题
https://github.com/facebook/nuclide/issues/232
https://github.com/facebook/nuclide/issues/201
https://github.com/facebook/nuclide/issues/199
https://github.com/facebook/nuclide/issues/84

总的来看，官方貌似没有提供什么有效的解决方案，根据有的同学的建议感觉从源码安装貌似有希望解决这个问题，试了一下发现果然可以，下面我介绍一下安装方法，不要怕，nuclide 提供的安装方法非常简单。

卸载nuclide

如果你没有安装过nuclide插件可以直接跳过这一步，如果你不幸已经通过Atom 安装过nuclide插件要先卸载。

你可以打开 ~/.atom/packages目录，然后删除所有nuclide开头的文件夹，还有一个hyperclick文件夹也是nuclide的插件，不要落下。

如果的Atom是刚刚安装的，你可以直接删除所有atom配置文件（请谨慎使用）。

`rm -Rf ~/.atom`

克隆出源代码

git clone https://github.com/facebook/nuclide.git

检查安装环境

Python 2.6 及以上.
Atom 0.209.0 及以上.
Node 0.12.0 及以上.

安装

15-12-28更新
今天发现安装方式发生了一些变化，之前很多单独包被整合成了一个包，可以使用以下步骤进行安装。

  1. npm install
  2. apm link

以下内容已过期:

来到源代码的根目录运行

./scripts/dev/setup

如果你使用Windows：

python scripts\dev\setup

重启

然后重启Atom，基本就搞定了。
